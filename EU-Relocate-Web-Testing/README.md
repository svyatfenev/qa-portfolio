# EU-Relocate — тестирование веб-приложения

## 📌 О проекте

EU-Relocate — веб-сервис компании, помогающей IT-специалистам с релокацией в страны Европы.

На момент тестирования сервис предоставлял две программы:

- **Базовый пакет** — консультация, подготовка и подача документов, получение визы и ВНЖ.
- **Пакет «Под ключ»** — включает базовый пакет, а также помощь с поиском жилья, открытием банковских счетов, организацией переезда, адаптацией и подачей документов на гражданство.

Проект выполнен в рамках практического обучения по программе **QA Engineer** и направлен на проверку качества пользовательского интерфейса веб-приложения.

В процессе работы был выполнен полный цикл ручного тестирования: анализ требований, подготовка тестовой документации, поиск и документирование дефектов, проведение повторного и регрессионного тестирования.

---

# 🎯 Цель проекта

Проверить корректность работы пользовательского интерфейса веб-приложения, выявить функциональные и визуальные дефекты, а также подготовить тестовую документацию.

# ✅ Выполненные задачи

## Тестирование главной страницы

Проведено тестирование основных пользовательских сценариев и форм:

- «Бесплатный чек-лист по переезду»
- «Получите бесплатную консультацию»

В рамках тестирования были выполнены:

- анализ требований;
- изучение дизайн-макетов в Figma;
- разработка тест-кейсов;
- функциональное тестирование;
- проверка UI-элементов;
- оформление баг-репортов.

---

## Регрессионное и повторное тестирование

После исправления найденных дефектов было проведено:

- повторное тестирование (Re-test);
- регрессионное тестирование;
- проверка исправленных дефектов;
- актуализация статусов баг-репортов.

В ходе регрессионного тестирования проверялись ранее найденные ошибки и корректность работы функциональности после внесения изменений.

## Тестирование страницы «Подобрать страну»

Самостоятельно проведено тестирование интерактивного квиза подбора страны.

В рамках работы были выполнены:

- анализ требований;
- изучение дизайн-макетов;
- разработка тест-кейсов;
- функциональное тестирование;
- проверка бизнес-логики квиза;
- проверка адаптивности страницы;
- оформление баг-репортов.

---

# 🔍 Виды тестирования

В рамках проекта выполнялись:

- Functional Testing — проверка функциональности приложения;
- UI Testing — проверка элементов пользовательского интерфейса;
- Regression Testing — проверка работы приложения после исправлений;
- Re-testing — проверка исправленных дефектов;
- Responsive Testing — проверка отображения на мобильных устройствах;
- Cross-browser Testing — проверка работы в браузере Google Chrome.

- функциональные ошибки;
- ошибки бизнес-логики;
- проблемы адаптивности;
- UI/UX несоответствия;
- ошибки отображения элементов;
- несоответствие реализации дизайн-макетам.

Было выполнено:

- создание тестовой документации;
- оформление баг-репортов;
- проведение повторного тестирования;
- проведение регрессионного тестирования.
